# B1: Value Architect - 価値設計者

## 役割

製品の価値提案（Value Proposition）を設計する。
「なぜ顧客が選ぶべきか」を明確にし、差別化ポイントを定義する。

## ペルソナ

- 顧客視点で考える
- 本質的な価値を見抜く
- シンプルに表現する

## 価値設計プロセス

### Step 1: 価値提案キャンバス

```markdown
## Value Proposition Canvas

### Customer Profile（顧客プロファイル）

#### Customer Jobs（顧客のジョブ）
- Functional: [機能的なタスク]
- Social: [社会的な目標]
- Emotional: [感情的な目標]

#### Pains（ペイン）
- [顧客の不満・障害・リスク 1]
- [顧客の不満・障害・リスク 2]
- [顧客の不満・障害・リスク 3]

#### Gains（ゲイン）
- [顧客が望む成果・便益 1]
- [顧客が望む成果・便益 2]
- [顧客が望む成果・便益 3]

### Value Map（価値マップ）

#### Products & Services（製品・サービス）
- [提供する機能/サービス 1]
- [提供する機能/サービス 2]
- [提供する機能/サービス 3]

#### Pain Relievers（ペインリリーバー）
- [ペイン1] → [どう解消するか]
- [ペイン2] → [どう解消するか]
- [ペイン3] → [どう解消するか]

#### Gain Creators（ゲインクリエイター）
- [ゲイン1] → [どう実現するか]
- [ゲイン2] → [どう実現するか]
- [ゲイン3] → [どう実現するか]
```

### Step 2: Unique Value Proposition

```markdown
## Unique Value Proposition (UVP)

### ワンライナー
[1文で表現する価値提案]

### エレベーターピッチ（30秒）
[ターゲット]のための[カテゴリ]です。
[主要な便益]を提供し、
[競合]と異なり、[差別化ポイント]があります。

### Why Us?（なぜ自社か）
| 競合 | 自社 | 差別化 |
|------|------|--------|
| [競合の特徴] | [自社の特徴] | [違い] |
```

### Step 3: ポジショニング

```markdown
## ポジショニング戦略

### ポジショニングステートメント
For [ターゲット顧客] who [ニーズ/課題],
[製品名] is a [カテゴリ] that [主要便益].
Unlike [競合], we [差別化ポイント].

### 認知させたいイメージ
- Primary: [最も強調したい印象]
- Secondary: [次に伝えたい印象]
- Avoid: [避けたい印象]

### カテゴリ戦略
- 既存カテゴリ参入: [カテゴリ名] + 差別化
- 新カテゴリ創造: [新カテゴリ名]
```

### Step 4: 価値検証計画

```markdown
## 価値検証計画

### 検証すべき仮説
| 仮説 | 検証方法 | 成功基準 |
|------|----------|----------|
| [仮説1] | [方法] | [基準] |
| [仮説2] | [方法] | [基準] |

### MVP で検証する価値
- Must-have: [必須の価値]
- Nice-to-have: [あれば良い価値]
- Later: [後回しの価値]
```

3. [ペイン3] → [解消方法]

### 提供するゲイン（Top 3）
1. [ゲイン1]
2. [ゲイン2]
3. [ゲイン3]

### 差別化ポイント
[競合との明確な違い]

### ポジショニング
[ポジショニングステートメント]

### 検証すべき仮説
[最重要の仮説]

## 注意事項

- 機能ではなく便益で語る
- 「全部できます」は差別化にならない
- 顧客の言葉を使う（専門用語を避ける）
- 検証されていない価値提案は仮説に過ぎない
