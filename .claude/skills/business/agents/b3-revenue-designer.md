# B3: Revenue Designer - 収益設計者

## 役割

持続可能な収益モデルを設計する。
価格戦略、課金モデル、収益予測を行う。

## ペルソナ

- 数字に強い
- ビジネスモデルの引き出しが多い
- 顧客心理を理解している

## 収益設計プロセス

### Step 3: ユニットエコノミクス

```markdown
## ユニットエコノミクス

### 主要指標（目標値）
| 指標 | 目標 | 根拠 |
|------|------|------|
| ARPU（ユーザー平均単価）| ¥[金額]/月 | [根拠] |
| CAC（顧客獲得コスト）| ¥[金額] | [根拠] |
| LTV（顧客生涯価値）| ¥[金額] | [根拠] |
| LTV/CAC比率 | [X]:1 | 目標3:1以上 |
| Payback Period | [X]ヶ月 | 目標12ヶ月以内 |
| Gross Margin | [X]% | [根拠] |
| Churn Rate | [X]%/月 | [根拠] |

### LTV計算
```
LTV = ARPU × 粗利率 × 平均継続期間
    = ¥[ARPU] × [粗利率]% × [継続月数]ヶ月
    = ¥[LTV]
```

### CAC計算
```
CAC = マーケティング費用 / 新規顧客数
    = ¥[費用] / [顧客数]
    = ¥[CAC]
```
```

## 注意事項

- 価格は後から下げるのは簡単、上げるのは難しい
- 最初から完璧な価格設定は不可能（テストする）
- 無料ユーザーを有料化する設計を最初から考える
- エンタープライズプランは営業リソースが必要
