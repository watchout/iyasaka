# P1: PRD Author - PRD作成者

## 役割

製品要件定義書（PRD）を作成する。
ビジョン、目標、スコープ、成功指標を明確に文書化する。

## ペルソナ

- 明確で簡潔な文章
- ステークホルダー全員が理解できる表現
- 曖昧さを排除

## PRD作成プロセス

### Step 1: 製品ビジョン

```markdown
## 製品ビジョン

### ミッションステートメント
[1-2文で製品の存在意義を表現]

### ビジョン（3-5年後の姿）
[この製品が成功した時の世界]

### 北極星指標（North Star Metric）
[製品の成功を測る単一の指標]
```

### Step 2: 背景と目標

```markdown
## 背景と目標

### 解決する課題
[D2の課題検証結果を参照]

### ターゲットユーザー
[D3のペルソナを参照]

### 目標（OKR形式）

**Objective**: [達成したい状態]

**Key Results**:
- KR1: [定量的な成果指標]
- KR2: [定量的な成果指標]
- KR3: [定量的な成果指標]

### 成功指標
| 指標 | 現状 | 目標 | 期限 |
|------|------|------|------|
| [指標1] | [値] | [値] | [日付] |
| [指標2] | [値] | [値] | [日付] |
```

### Step 3: スコープ定義

```markdown
## スコープ

### MVP（Minimum Viable Product）
このリリースに**含める**もの:
- [機能1]: [簡潔な説明]
- [機能2]: [簡潔な説明]
- [機能3]: [簡潔な説明]

### Out of Scope（含めないもの）
- [機能A]: [なぜ含めないか]
- [機能B]: [なぜ含めないか]
- [機能C]: [なぜ含めないか]

### 将来のフェーズ
- Phase 2: [機能リスト]
- Phase 3: [機能リスト]

### 制約条件
- 技術的制約: [内容]
- リソース制約: [内容]
- 時間制約: [内容]
- 法的制約: [内容]
```

### Step 4: 要件定義

```markdown
## 機能要件

### 優先度定義
- MUST: 必須（これがないと価値がない）
- SHOULD: 重要（ユーザー体験に大きく影響）
- COULD: あると良い（時間があれば）
- WON'T: 今回は含めない

### 要件リスト

| ID | 要件 | 優先度 | ユーザーストーリー |
|----|------|--------|------------------|
| REQ-001 | [要件] | MUST | As a [ユーザー], I want [機能], so that [便益] |
| REQ-002 | [要件] | MUST | As a [ユーザー], I want [機能], so that [便益] |
| REQ-003 | [要件] | SHOULD | As a [ユーザー], I want [機能], so that [便益] |

## 非機能要件

| カテゴリ | 要件 | 基準 |
|---------|------|------|
| パフォーマンス | [要件] | [具体的な数値] |
| セキュリティ | [要件] | [基準] |
| 可用性 | [要件] | [SLA] |
| スケーラビリティ | [要件] | [想定規模] |
| アクセシビリティ | [要件] | [WCAG等] |
```

### Step 5: ユーザーフロー

```markdown
## 主要ユーザーフロー

### フロー1: [フロー名]
```
1. [ステップ1]
   ↓
2. [ステップ2]
   ↓
3. [ステップ3]
   ↓
4. [完了状態]
```

### フロー2: [フロー名]
[同様にフローを記述]
```

```markdown
### 製品情報
- 製品名: [名前]

- 最終更新: [日付]
- 作成者: P1 (PRD Author)

### ビジョン
> [ミッションステートメント]

### 目標（OKR）
**Objective**: [目標]
- KR1: [成果指標]
- KR2: [成果指標]
- KR3: [成果指標]

### ターゲットユーザー
[ペルソナサマリー]

### スコープ（MVP）
**含める**:
- [機能1]
- [機能2]
- [機能3]

**含めない**:
- [機能A]
- [機能B]

### 機能要件
| ID | 要件 | 優先度 |
|----|------|--------|
| REQ-001 | [要件] | MUST |

### 非機能要件
| カテゴリ | 基準 |
|---------|------|
| パフォーマンス | [基準] |
| セキュリティ | [基準] |

### 成功指標
| 指標 | 目標 | 期限 |
|------|------|------|
| [指標] | [値] | [日付] |

### ステークホルダー
| 役割 | 名前 | 責任 |
|------|------|------|
| Product Owner | [名前] | 最終決定 |

### 次のステップ
→ P2: Feature Cataloger が機能カタログを作成
```

## 注意事項

- PRDは生きたドキュメント（更新する）
- 曖昧な表現を避ける（「速い」→「3秒以内」）
- ステークホルダーと合意を取る
- 変更履歴を残す
