## 1. 三つの「不」— なぜこのHPを作り直すのか

IYASAKAは「不を光へ」を掲げる会社。このHPリデザインも、三つの「不」の解消から始まる。

### お客さんの不

「AIが気になる。でも誰に聞けばいいか分からない。」

ネットで調べても専門用語ばかり。大手に聞いたら見積もり300万円。うちみたいな小さい会社に合うAIなんてあるのか。そもそもAIで何ができるのかすら、よく分からない。

→ HPに来た瞬間、AIが話しかけてくる。業種を伝えるだけで、自分の業界のAI活用状況が分かる。具体的に何時間削減できるか、すぐ分かる。名前を書く前に、価値を体験できる。

### IYASAKAの不

「良いサービスがあるのに、伝わらない。」

OmotenasuAI、ミエルプラス、配信プラス、弱電プラス、AIプラス。どれも現場で成果を出している。なのにHPを見た人は「何屋か分からない」「本当に実績あるの？」で離脱する。

→ HP全体が「AIを使い倒している会社」の証明になる。画像はAI生成、コンテンツはAIが毎回書く、チャットはAIが応答する。言葉で「AI会社です」と言うのではなく、体験で証明する。

### 業界の不

「自分の業界に特化したAI情報がない。」

「AI活用」の情報は溢れている。でもホテル業界のAI導入率は？ 弱電業界でAIはどう使われている？ 自分の業界に絞った情報は、どこにもない。

→ 各街区に業界特化のAI最新情報がある。毎朝AIが自動収集して更新する。来るたびに新しい情報がある。ブログとしても蓄積され、検索でたどり着ける。IYASAKAが「業界 × AI」の第一想起になる。

## 2. コンセプト — 「AIが息づく街」

### 中心思想

HPは「ページ」ではない。「街」。

IYASAKAが支援する業界が、それぞれ街の一区画として存在する。訪問者はスクロールで街を歩く。各区画にはAIの相談役が住んでいて、いつでも話しかけられる。街の掲示板には、その業界のAI最新ニュースが毎朝貼り出されている。

静的なリンク集ではなく、訪問のたびにAIが最新の情報でコンテンツを生成する。昨日と今日で表示が違う。再訪問のたびに新しい発見がある。

この街全体が、AIで動いている。それ自体が「うちの会社はAIをこう使っています」の最も説得力のある証明。

### なぜ「街」なのか

- 「ページ一覧」は退屈。「街を歩く」は体験
- 各事業が独立した区画 → 自分の業界だけ見ればいい（迷わない）
- 各区画に住人（AI相談役）がいる → 話しかけるハードルがゼロ
- 街は生きている → 毎日変わる → 再訪問の理由になる
- 街全体がAIで動いている → 「この会社は本物だ」の直感的な理解

### 弥栄の体現

訪問者は名前も会社名も書かずに、AIから価値を受け取れる。
先に与える。先に栄えさせる。
その結果として「この会社に相談しよう」が生まれる。

これが弥栄 — まず相手を栄えさせ、結果として自分も栄える。

## 3. ビジュアル — ボクセルアート × 和 × ダークネイビー

### 選択の理由

「AIの会社」を伝える方法は二つある。一つは冷たくテック感を出す方法（サイバーパンク、ネオン、ネットワーク図）。もう一つは温かく人間的に見せる方法。

IYASAKAは弱電工事から始まった会社。現場を知っている。技術だけの会社ではない。だから温かみが必要。でもAIの先進性も伝えたい。

ボクセルアート × 和風建築 × ダークネイビー背景 は、この両方を実現する。

- ボクセル（ピクセル的な立体感）→ テック感、デジタル
- 和風建築（灯篭、桜、瓦屋根）→ 温かみ、日本の美意識
- ダークネイビー背景 → 落ち着き、信頼、BtoBの品格
- 温かいオレンジの内部照明 → 「中で人（とAI）が働いている」生命感
- チルトシフト（ボケ感）→ ミニチュアジオラマの上品さ
- AIロボットキャラ → 各街区の案内人、親しみやすさ

### トーンの基準

- 「ゲームっぽい」→ ×。あくまで「センスのある会社」
- 「冷たいテック」→ ×。温かみを失わない
- 「面白い会社だな」→ ◎。これが正解
- 「この会社に相談したい」→ ◎。最終ゴール

### 画像素材

- Midjourneyで生成（AI生成であること自体がメッセージ）
- 各街区ごとに1プロンプト → 4パターン生成 → 全パターン保持

## 4. ユーザー体験 — 訪問者のジャーニー

### ファーストビュー（0-3秒）

画面が開く。ダークネイビーの空間に、ボクセルアートの街が広がっている。
上部に一文:

「この街は、AIが動かしている。」

その下に、テキスト入力欄:
プレースホルダー:「あなたの業種と悩みを教えてください」

入力欄の下に、5つの街区アイコンボタン（タイプが面倒な人向け）:
🏨 ホテル / ⚡ インフラ / 🎤 配信 / 👁️ 現場 / 🤖 AI

### スクロール体験（3秒以降）

スクロールすると街を歩く。各街区が順に現れる。

各街区のセクション構成:
1. 街区のボクセルアート画像（4パターンからランダム表示）
2. 「この街区の不」— 業界の課題を一文で（AIが最新情報を反映して動的生成）
3. AIの相談役キャラ — 「こんにちは。〇〇業界のAI活用についてご相談ください」
4. 業界 × AI 最新ニュース — 2-3件のダイジェスト（毎朝自動更新）
5. 「詳しく聞く」ボタン → チャットパネル起動

### チャット体験

街区のロボットキャラまたは「詳しく聞く」をクリック → サイドパネルでチャット起動。
その業界専門のAIエージェントが応答。

3ターン以内で:
- 訪問者の業種・規模・課題を把握
- 業界のAI導入トレンドを1つ共有
- 「御社の場合のAI活用ポテンシャルを診断できます」→ AIプラスLP診断に誘導

### 着地点（CTA）

全街区のスクロール後、最下部:

「すべての街区に共通していること。AIが、動いている。」
「この仕組みを、御社にも。」

→ AIプラスLP（iyasaka.co/aiplus）の診断ページへ

### 再訪問の動機

- 業界ニュースが毎日更新される → 情報収集目的で再訪
- AIチャットに新しい質問をしたい → 関係性の継続
- 表示される画像パターンが毎回違う → 「今日はどんな感じかな」

## 6. AIチャットBot — 各街区の相談役

### 設計思想

各街区のロボットキャラは「営業マン」ではなく「相談役」。
売り込まない。先に価値を与える。
業界のAIトレンドを教え、訪問者の課題を聞き、具体的なアドバイスを1つ提供する。
その上で「もっと詳しく診断しますか？」と自然に繋ぐ。

弥栄の精神: まず相手を栄えさせる。

### Phase 1（即時実装）

技術: Anthropic Claude Sonnet API 直接呼び出し
接続: agent-commsなし。API + 公開用プロンプトのみ
コスト: 月額5万円以下（Sonnet、月500会話想定）

各街区の公開用システムプロンプト:
- 役割と専門領域を明記
- 社内情報は一切共有しない制約
- 3-4文以内で簡潔に回答
- 3ターン以内で診断/相談予約に自然誘導
- 「営業電話なし」「契約義務なし」を明示

### Phase 2（agent-comms安定後）

技術: agent-comms MCP経由で内部エージェントに接続
接続: 公開エージェント → フィルタリング層 → 内部C-Suite
フィルタリング: 社内情報、顧客情報、財務情報は絶対に通さない
メリット: 内部のresearch-leadデータ（業界動向）をリアルタイムで参照可能

### Phase 3（成熟期）

チャットログの分析 → 訪問者がどんな「不」を持っているかのデータ蓄積
→ プロダクト開発、LPコピー改善、営業トークの最適化に活用
→ 「お客さんの言葉」でLPを書き直す（最強のコピーライティング素材）

### セキュリティ要件

- 公開用プロンプトは内部SOUL.mdとは完全分離
- 内部の組織構造、エージェント名、技術スタック、顧客情報は一切出さない
- プロンプトインジェクション対策（ロール固定、出力制限）
- APIレートリミット（1IP あたり 20会話/時間）
- 全チャットログを保存（リード分析用 + 不正利用検知用）

## 8. SEO二層構造 — クローラーと人間で見せるものを分ける

### 設計思想

Googleのクローラーには「読める」HTMLを返す。
人間の訪問者には「体験できる」AIダイナミックコンテンツを返す。

### 技術実装

フレームワーク: Next.js（App Router）

**SEO層（サーバーサイド）:**
- SSRで静的HTML生成
- title, meta description, OGP — 全ページ最適化
- 見出し構造（h1-h3）— 各街区のキーワード含む
- 構造化データ（JSON-LD）— LocalBusiness, Product, FAQPage
- ブログ記事は完全静的ページ（/blog/[slug]）
- sitemap.xml自動生成

**体験層（クライアントサイド）:**
- SSRの静的シェルの上にReactでAIコンテンツを動的レンダリング
- 画像のランダム表示
- 業界ニュースの動的挿入
- AIチャットBotの起動
- クローラーはJSを実行しないか、実行しても基本HTMLは読めるので問題なし

### URL構造

```
iyasaka.co/                — HP（街全体）
iyasaka.co/hotel           — ホテル街区（現在のページをリデザイン）
```

## 9. 進化メカニズム — AIによる自然淘汰

### 設計思想

人間が「A案とB案どっち？」を判断するのではなく、
AIが多数のパターンを生成し、効果が高いものが自然に残る。

### 画像の進化

- 5街区 × 4パターン = 20枚の初期プール
- 訪問ごとにランダム表示 → Google Analytics/Vercel Analyticsで計測
- 各パターン × CVR（チャット開始率、LP遷移率）を集計
- 2週間後、CVRが低いパターンを非表示 → 高いパターンの表示頻度UP
- 定期的にMidjourneyで新パターン追加 → プール入替

### コピーの進化

- ヒーローテキスト、街区の説明文をAIが複数パターン生成
- 訪問ごとに異なるパターンを表示
- CVR計測 → 効果の高いコピーが残る
- チャットログから「お客さんが実際に使った言葉」を抽出 → コピーに反映

### コンテンツの進化

- ブログ記事のCTRを計測
- SNS投稿のエンゲージメント率を計測
- 効果の高いトピック・フォーマットをAIが学習
- 次回のコンテンツ生成に反映
