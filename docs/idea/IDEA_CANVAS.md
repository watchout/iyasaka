## 2. 課題（Problem）

### 2.1 解決したい課題

**主要課題（1つだけ）**:
```
中小企業の経営者・現場責任者が「現場の不（痛み）」を抱えながら、
適切な相談先・ソリューションを見つけられない孤独。
```

**関連する課題**:
1. 現場の情報がホワイトボードやExcelに散在し、リアルタイムに把握できない
2. IT担当者不在のため、トラブル発生時に頼れる専門家がいない
3. AIやDXツールを導入しても形骸化し、現場に定着しない

### 2.2 課題の深掘り

| 質問 | 回答 |
|------|------|
| この課題は誰が持っている？ | 従業員3-100名の中小企業経営者、現場責任者 |
| どんな状況で発生する？ | 設備トラブル、人手不足、情報管理の破綻、業務効率化の失敗時 |
| どのくらいの頻度で発生する？ | 日常的（特に建設・製造・物流・宿泊業で顕著） |
| 発生すると何が困る？ | 業務停止、収益機会の損失、スタッフの疲弊、経営判断の遅延 |
| 今はどう対処している？ | Excel/ホワイトボード、都度見積の業者、汎用ツールの無理な適用 |
| 対処にいくらかかっている？ | 専門業者の都度見積（高額・不透明）、人件費の浪費 |

### 2.3 課題の検証状況

| 検証方法 | 実施 | 結果 |
|---------|------|------|
| 自分自身が課題を経験 | 済 | 弱電工事・ホテルDX・現場管理で実体験あり |
| 身近な人にヒアリング（5人以上） | 済 | PRODUCT_PAIN_POINTS.md に記録（6製品分） |
| SNS/コミュニティでの反応確認 | 一部 | 中小企業DX関連の課題は広く共有されている |
| 既存サービスのレビュー分析 | 済 | 競合分析: Excel/汎用ツール/大手SIer vs IYASAKA |
| 検索ボリューム確認 | 一部 | 「中小企業 DX」「業務効率化」等で需要確認 |

## 3. ソリューション（Solution）

### 3.1 提供する解決策

```
ナラティブ駆動のコーポレートサイトで「不 -> 光」の物語を通じて
中小企業の経営者に共感し、診断 -> 無料相談 -> 製品導入の導線で
適切なソリューション（5つの「プラス」製品）へ接続する。
```

### 3.2 主要機能（MVP = Phase 1）

| # | 機能 | 課題との関連 | 優先度 |
|---|------|-------------|-------|
| 1 | コーポレートサイト（トップ・会社概要・サービス一覧） | ブランド認知・信頼構築 | P0 |
| 2 | 製品別LP（5製品 + AIプラス診断） | リード獲得の主要チャネル | P0 |
| 3 | 共通問い合わせフォーム（不 -> ソリューション マッピング） | コンバージョン | P0 |
| 4 | AIEO対応（JSON-LD / FAQ / 構造化データ） | AI検索エンジンからの流入 | P0 |
| 5 | リードトラッキング（Plausible + UTM） | 計測・改善サイクル | P0 |

### 3.3 将来の機能（Phase 2以降）

| # | 機能 | 実装時期 |
|---|------|---------|
| 1 | HP SaaS化（マルチテナント対応） | Phase 2 (3-6週) |
| 2 | 管理画面（ページ編集・リード管理・CSV出力） | Phase 2 |
| 3 | AIコンテンツ生成 | Phase 3 (7週-) |
| 4 | Stripe課金連携 | Phase 3 |
| 5 | 業種パック（電気工事テンプレート等） | Phase 3 |

## 5. 差別化（Differentiation）

### 5.1 なぜIYASAKAが作るのか？

| 質問 | 回答 |
|------|------|
| この領域の経験・知識 | 弱電工事・ホテルDX・現場管理の実務経験 |
| ユニークな視点 | 「不 -> 光」のナラティブ設計、現場第一主義 |
| 他にない強み | 製品横断のブランド一貫性、JV/サブスク型リスク分担モデル |
| なぜ今このタイミング？ | 2026年: 中小企業の人手不足が過去最高水準、IT格差拡大中 |

### 5.2 競合との違い

| 比較軸 | Excel/ホワイトボード | 汎用SaaS | 大手SIer | IYASAKA |
|-------|---------------------|----------|---------|---------|
| 導入コスト | 低 | 中 | 高 | 中（サブスク/JV） |
| 現場適合 | 低（手作業） | 低（汎用） | 中 | 高（業種特化） |
| 伴走支援 | なし | なし | 有（高額） | 有（リスク分担型） |
| **差別化** | - | - | - | **現場に寄り添うパートナー** |

### 5.3 参入障壁（Moat）

```
- 実体験ベースのペインポイント理解（6製品分のユーザーヒアリング済み）
- ナラティブOS（整える・つなぐ・育てる）による一貫したブランド体系
- JV/サブスク型ビジネスモデルによる低リスク導入
- dev-OS品質基盤による全製品の品質保証
```

### 6.1 収益モデル

| モデル | 採用 | 詳細 |

Phase 1では直接課金なし。HPは営業ツールとしてリード獲得に専念。

## 8. リスクと前提

### 8.1 主要なリスク

| リスク | 影響度 | 対策 |
|-------|-------|------|
| リード獲得が想定を下回る | 高 | 診断導線型CRO、A/Bテスト前提の設計 |
| HP SaaS化の技術的複雑性 | 中 | Phase 1で灯台テナント検証後にPhase 2へ |
| 製品数過多による焦点の分散 | 中 | HP表示優先度の明確化（ミエル > 弱電 > 他） |

### 8.2 検証すべき前提

| 前提 | 検証方法 | 検証時期 |
|------|---------|---------|
| 診断型CTAでCVR 8%以上が実現可能か | LP + 診断フォーム計測 | Phase 1 |
| AIEO対応でAI検索からの流入が得られるか | 構造化データ実装 + 計測 | Phase 1 |
| HP SaaS化で2社目テナントが再現できるか | 東海電業社での検証 | Phase 2 |
