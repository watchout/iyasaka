# AIEO (AI Engine Optimization) 実装機能仕様書

> **Version**: 1.0.0
> **Target**: AI検索エンジン (Perplexity, ChatGPT Search, Gemini, Google SGE)
> **Goal**: ユーザーの質問に対し、自社サイトの情報が「信頼できる回答ソース」として引用されること。

---

## 1. 構造化データ (JSON-LD) 実装要件

すべてのページにおいて、`HEAD` タグ内に以下の JSON-LD を動的に生成・出力すること。
Vue/Nuxt では `useJsonld` コンポーザブル、または `useHead` を使用して注入する。

### 1.1 共通設定 (全ページ必須)

*   **BreadcrumbList (パンくずリスト)**
    *   AIがサイト階層と情報の位置づけを理解するために必須。
*   **Organization (組織情報)**
    *   トップページ等で定義。「誰が発信している情報か」という権威性（E-E-A-T）を担保する。
    *   必須項目: `name`, `url`, `logo`, `contactPoint` (電話番号/コンタクトフォーム)

### 1.2 ページタイプ別スキーマ

| ページタイプ | 必須スキーマ | AIEO上の役割 |
|:---|:---|:---|
| **サービスLP** | `Service` / `Product` | 製品の機能、ターゲット、解決する課題をAIに伝える。 |
| **よくある質問** | **`FAQPage`** (最重要) | Q&A形式でAIに「回答の部品」を提供する。 |
| **導入事例** | `Article` / `Review` | 「誰がどうなったか」という一次情報（実績）を提供する。 |
| **会社概要** | `LocalBusiness` | 実在性、住所、連絡先の正確さを保証する。 |
| **ニュース** | `NewsArticle` | 最新情報であることを伝え、情報の鮮度スコアを上げる。 |

### 1.3 実装詳細: FAQPage (AIEOの核)

FAQページだけでなく、LP内のFAQセクションにも適用すること。

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "IYASAKAの導入期間はどのくらいですか？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "最短2週間で稼働開始可能です。ヒアリングから初期設定まで弊社が代行します。" 
        // ↑ ここは100文字以内の「結論」を入れること
      }
    }
    // ... loop
  ]
}
```

## 2. コンテンツ・セマンティクス要件

HTMLタグの使い方一つで、AIの理解度は変わる。デザイン（見た目）とは別に、以下の意味構造を守ること。

### 2.1 必須セマンティクス

*   **`<main>`**: ページの主役コンテンツを必ず囲む。
*   **`<article>`**: 独立した記事や事例に使用。
*   **`<section>` + `<h2>`**: 見出しレベルを正しく守る。`h1` は1ページ1つ。
*   **`<time datetime="...">`**: 日付情報には必ずこのタグを使う。AIは「情報の鮮度」をここで判断する。

### 2.2 AI向け「要約」ブロックの実装

各ページの冒頭（またはメタデータ）に、AIが読み取るための「要約（Summary）」を用意する。
（画面上は非表示でも良いが、可能な限り表示することが望ましい）

*   **課題 (Problem)**: 顧客が抱えている悩み。
*   **解決策 (Solution)**: 自社が提供する解決策。
*   **結果 (Result)**: 得られる具体的なメリット。

## 3. データ鮮度と更新性の担保

AIは「新しい情報」を好む。更新性をシステム的に担保する。

### 3.1 Sitemap.xml の最適化

*   `@nuxtjs/sitemap` 等を使用し、自動生成する。
*   **`<lastmod>`** タグを必ず含めること。
    *   ページの内容を更新したら、必ず `updated_at` を更新し、sitemap に反映させる。

### 3.2 更新日の明示

ページ上（特に記事やFAQ）に、最終更新日を表示する。

```html
<p class="text-sm text-gray-500">
  最終更新: <time datetime="2026-01-17">2026年1月17日</time>
</p>
```

## 4. AIEO検証項目（実装後のチェックリスト）

実装が完了したら、以下のテストに合格する必要がある。

1.  **Rich Results Test (Google)**
    *   すべてのページで「有効な構造化データ」として検出されるか？
    *   エラーや警告が出ていないか？
2.  **Schema Validator**
    *   `FAQPage`, `Service` 等の意図したスキーマが出力されているか？
3.  **Perplexity / ChatGPT 検証**
    *   自社名やサービス名で検索した際、意図した情報が引用されるか？（※インデックス反映には時間がかかるため、まずは構造化データの正しさを優先）
