# IYASAKA 共通問い合わせフォーム仕様書

> 全プロダクトLPからの流入を受け止める「母艦」としてのフォーム設計。
> エモーショナルな選択肢で「不」を言語化し、適切なソリューションへ導く。

---

## 1. フォーム設計コンセプト

### ゴール
- ユーザーの「不」を言語化させ、共感を生む
- 製品カテゴリ（整える/つなぐ/育てる）に応じた適切なヒアリング
- 送信後のクロスセルで「弥栄循環」を促進

### UX原則
- **段階的開示**: 最初は最小限、興味に応じて詳細を表示
- **共感ファースト**: 専門用語を避け、感情に寄り添う表現
- **摩擦の最小化**: 必須項目は最小限に

### Step 4: 詳細ヒアリング（製品カテゴリ別・条件分岐）

#### 【整える】カテゴリ選択時

```
現在の課題について教えてください：

□ 在庫の数が合わない・探す時間が多い
□ 予定がホワイトボード頼み・共有できていない
□ 車両の位置や稼働状況が把握できない
□ 図面や資料がどこにあるか分からない
□ ネットワーク・設備のトラブルが頻発

現在の管理方法：
○ 紙・ホワイトボード
○ Excel・スプレッドシート
○ 既存システムあり（乗り換え検討）
○ 特に管理していない

従業員規模：
○ 1〜10名
○ 11〜50名
○ 51〜100名
○ 101名以上
```

#### 【つなぐ】カテゴリ選択時

```
配信・発信について教えてください：

□ イベントやセミナーを配信したい
□ 社内向けの情報発信を強化したい
□ 海外向けにコンテンツを展開したい
□ 既存の動画資産を活用したい

配信の頻度（予定）：
○ 月1回未満
○ 月1〜4回
○ 週1回以上
○ 毎日

現在の配信環境：
○ まったくない（これから）
○ 簡易的な機材はある
○ 本格的なスタジオがある
```

#### 【育てる】カテゴリ選択時

```
AI活用について教えてください：

□ ChatGPT等を試したが定着しない
□ 接客の質を均一化・向上させたい
□ データを活かした意思決定をしたい
□ 業務効率化のアイデアが欲しい

AI導入経験：
○ なし（初めて）
○ 一部導入済み
○ 全社的に活用中（改善したい）

最も期待する効果：
○ 時間の創出
○ 売上向上
○ 顧客満足度向上
○ スタッフの負担軽減
```

## 4. データ構造

### リードテーブル（Supabase）

```sql
CREATE TABLE leads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  
  -- 基本情報
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  company TEXT,
  phone TEXT,
  
  -- 「不」の選択（配列）
  pain_points TEXT[],
  
  -- 製品選択
  product_id TEXT,
  product_category TEXT, -- organize/connect/nurture
  
  -- 詳細ヒアリング（JSONB）
  survey_responses JSONB,
  
  -- 自由記述
  message TEXT,
  
  -- トラッキング
  origin_product_id TEXT,  -- p_id パラメータ
  utm_source TEXT,
  utm_campaign TEXT,
  utm_medium TEXT,
  
  -- ステータス
  status TEXT DEFAULT 'new', -- new/contacted/converted/lost
  assigned_to TEXT,
  
  -- メタデータ
  ip_address TEXT,
  user_agent TEXT
);

-- インデックス
CREATE INDEX idx_leads_status ON leads(status);
CREATE INDEX idx_leads_product ON leads(product_id);
CREATE INDEX idx_leads_created ON leads(created_at DESC);
```
