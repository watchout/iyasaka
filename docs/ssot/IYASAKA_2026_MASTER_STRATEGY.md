## 1. エグゼクティブサマリー

### 1.1 プロジェクト概要

| 項目 | 内容 |
|------|------|
| **コンセプト** | 「今ある『不』を、未来の『光』へ」 |
| **ターゲット** | 従業員3〜100人規模の中小企業、現場責任者、経営者 |
| **主要課題（2026年市場）** | 深刻な現業職の人手不足、IT格差による中小企業の収益停滞、管理不全のリスク拡大 |
| **ソリューション** | 5つの「プラス」製品 + dev-OS基盤 |

### 1.2 目標KPI

| 指標 | 目標値 |
|------|--------|
| CVR（診断開始率） | **8%以上** |
| リード獲得数 | **月間150件以上** |
| Lighthouse Score | **90以上（全デバイス）** |

### 1.3 戦略的意思決定ルール

フィードバック統合時の優先順位：

1. **優先度1**: CVRが落ちる可能性のある演出は全て後置（下層）
2. **優先度2**: FVは具体的痛み訴求、2段目でナラティブ回収
3. **優先度3**: 速度（LCP/INP/CLS）を最優先のガードレール
4. **優先度4**: 判断が割れたら、A/Bテスト前提で両案を残す

## 2. 市場分析と戦略的位置づけ

### 2.1 2026年の市場環境

| 危機 | 内容 |
|------|------|
| **危機1: 人手不足** | 中小企業の人手不足感：過去最高水準。特に建設・製造・物流・サービス業で顕著。 |
| **危機2: IT格差** | 中小企業の8割がIT導入済みだが成果実感なし。大企業との利益格差が拡大中。 |
| **危機3: 管理不全** | IT担当者不在の企業：35%。「止まったら終わり」というインフラへの恐怖。 |

### 2.2 IYASAKAの差別化ポイント

| 競合・代替 | 弱点 | IYASAKAの差分 |
|-----------|------|---------------|
| Excel/ホワイトボード | 入力負担、二重管理 | 自動取得・可視化（ミエルプラス） |
| 汎用グループウェア | 現場に合わない、形骸化 | 業種特化・伴走支援（AIプラス） |
| 都度見積の工事屋 | 緊急時に高額、予防なし | サブスク保守（弱電プラス） |
| 汎用チャットボット | 定型回答のみ、連携なし | PMS/CRM連携（OmotenasuAI） |
| 高額外注配信会社 | 投資リスク大 | JVでリスク分担（配信プラス） |

**共通の強み**: JV・サブスク・伴走という「リスク分担型」ビジネスモデル

### 4.2 診断ロジック

```javascript
// 5問の質問
const questions = [
  { id: 'q1', text: '「現場の状況が見えない」不安がありますか？', product: 'mieru-plus' },
  { id: 'q2', text: 'システムやインフラが「止まったら終わり」という恐怖はありますか？', product: 'jakuden-plus' },
  { id: 'q3', text: '人手不足で接客・対応の質が下がっていませんか？', product: 'omotenasu-ai' },
  { id: 'q4', text: '収益源を多角化したいが、投資リスクが怖いですか？', product: 'haishin-plus' },
  { id: 'q5', text: 'AIやDXツールを導入したが、使われていませんか？', product: 'ai-plus' }
];

// 分岐ロジック: 最初の「はい」を優先
const diagnose = (answers) => {
  for (const q of questions) {
    if (answers[q.id] === 'yes') return q.product;
  }
  return 'mieru-plus'; // 全てNoの場合
};
```

## 5. デザインシステム（Neo-Japanesque）

### 5.1 コンセプト

**Neo-Japanesque = 最先端Web × 日本の伝統美**

- 「余白」が生む品格
- 「非対称」が生む動き
- 「暁光」が生む希望

### 5.2 カラーパレット

```css
:root {
  /* プライマリー */

  --color-akatsuki: #FF6B35;      /* 暁光オレンジ（CTA） */
  
  /* セカンダリー */
  --color-ash: #8B8B8B;           /* 灰色（補助テキスト） */
  --color-pale-orange: #FFE8D6;   /* 淡いオレンジ（ホバー） */
  --color-deep-sumi: #0D0D0D;     /* 濃墨（強調） */
  
  /* システムカラー */
  --color-success: #2D6A4F;       /* 成功（和の緑） */
  --color-warning: #D4A574;       /* 警告（和の金） */

```
