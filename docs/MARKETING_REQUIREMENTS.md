# IYASAKA マーケティング機能要件書

**作成日**: 2024年12月18日  
**目的**: HPを「オンライン上の名刺代わり」として機能させ、信頼性・権威性を確立し、間接的なコンバージョンを促進する  
**対象閲覧者**: 顧客/顧客候補、採用候補者、メディア/提携企業

---

## 🎯 HPの戦略的役割

1. **信用の証明** - オンラインプレゼンスを高め、企業の信頼性・権威を示す
2. **取りこぼし防止** - 本来顧客となるはずの人の離脱を防ぐ
3. **間接的コンバージョン** - 直接的な集客ではなく、信頼醸成を通じた成約率向上
4. **情報発信基地** - メディアとの接点を作り、専門性を客観的に示す

---

## I. 信頼性と権威性を確立するための要素

### 1. 具体的なプロフィール情報の公開

**目的**: 信頼性の確立、メディアからの取材フック提供

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| 代表者経歴 | 出身地、学歴、職歴、過去の勤務先を「政治家並み」に詳細記述 | 🔴 高 |
| 主要メンバー紹介 | 技術責任者、営業責任者など主要メンバーの経歴 | 🟡 中 |
| 固有名称の明記 | 「様々な...」ではなく具体的な会社名・役職を記載 | 🔴 高 |

**実装ページ**: `/company` または `/about`

**データ構造案**:
```typescript
interface TeamMember {
  name: string
  role: string
  photo?: string
  birthplace?: string        // 出身地
  education: string[]        // 学歴（学校名、卒業年）
  career: CareerEntry[]      // 職歴（会社名、役職、期間）
  achievements?: string[]    // 主な実績
  message?: string           // 一言メッセージ
}
```

---

### 2. 創業ストーリーと挫折エピソード

**目的**: 人間的な魅力と親近感の醸成、メディアの興味喚起

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| 創業背景 | なぜこの事業を始めたのか（動機・きっかけ） | 🔴 高 |
| 挫折エピソード | 困難をどう乗り越えたか（浮き沈みのあるストーリー） | 🔴 高 |
| ビジョン | 今後どこを目指しているのか | 🟡 中 |

**実装ページ**: `/company/story` または `/about/story`

**コンテンツ構成案**:
1. 創業前（何をしていたか、何に課題を感じていたか）
2. 創業のきっかけ（転機となった出来事）
3. 挫折・困難（具体的なエピソード）
4. 乗り越えた方法（学びと成長）
5. 現在と未来（今のビジョン）

---

### 3. 顧客事例（実績）の充実と強調

**目的**: 客観的な事実による信頼性向上、メディア取材のきっかけ提供

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| ビフォーアフター | 導入前後の具体的な変化（数値含む） | 🔴 高 |
| 業種別事例 | 様々な業態での成功事例 | 🔴 高 |
| 顧客の声 | 実際の利用者の感想・評価 | 🔴 高 |
| 数値実績 | CVR向上率、コスト削減額など | 🟡 中 |

**実装ページ**: `/cases` および `/cases/[slug]`

**データ構造案**:
```typescript
interface CaseStudy {
  slug: string
  title: string
  client: {
    name: string
    industry: string
    size?: string
    logo?: string
  }
  challenge: string          // 導入前の課題
  solution: string           // 提供したソリューション
  results: {
    metric: string           // 指標名
    before: string           // 導入前
    after: string            // 導入後
    improvement: string      // 改善率
  }[]
  testimonial?: {
    quote: string
    author: string
    role: string
  }
  products: string[]         // 関連プロダクト
  publishedAt: string
}
```

### 4. メディア掲載実績のロゴ/リンク掲載

**目的**: 「まともな会社」であることの証明、競合との差別化

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| メディアロゴバー | HPの目立つ場所にロゴを横並び表示 | 🔴 高 |
| 掲載記事リンク | 実際の記事へのリンク | 🟡 中 |
| 掲載日・媒体名 | 「2024年10月 日経新聞掲載」など | 🟡 中 |

**実装場所**: 
- トップページ Hero下部
- `/company` ページ
- `/press` ページ（メディア掲載一覧）

**データ構造案**:
```typescript
interface MediaCoverage {
  id: string
  mediaName: string          // 媒体名
  mediaLogo?: string         // ロゴ画像
  mediaType: 'tv' | 'newspaper' | 'magazine' | 'web' | 'radio'
  title: string              // 記事タイトル
  url?: string               // 記事URL
  publishedAt: string        // 掲載日
  featured: boolean          // トップページに表示するか
}
```

## II. メディアと提携企業へのアクセスを促す要素

### 5. 「メディア関係者様へ」専用ページ

**目的**: メディアからの取材依頼をスムーズにする（99%の企業が持っていない差別化要素）

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| 取材可能テーマリスト | 専門家として語れるテーマ一覧 | 🔴 高 |
| プレスキット | 企業ロゴ（高解像度）、代表者写真のダウンロード | 🔴 高 |
| 取材対応フロー | 問い合わせから取材までの流れ | 🟡 中 |
| FAQ | よくある取材質問への回答 | 🟡 中 |
| 直接連絡先 | メディア専用の問い合わせ先 | 🔴 高 |

**実装ページ**: `/press` または `/media`

**コンテンツ構成案**:
```markdown
# メディア関係者様へ

## 取材可能なテーマ
- 弱電工事業界のDX動向
- ホテル業界のAI活用最前線
- 中小企業の配信ビジネス参入
- ...

## プレスキット
- 企業ロゴ（PNG/SVG/AI）
- 代表者写真（高解像度）
- 会社概要資料（PDF）
- 事業説明資料（PDF）

## 取材のお問い合わせ
- 担当: 広報担当
- Email: press@arrowsworks.com
- 電話: 048-872-6822
```

### 6. コラム/ブログ（読み物）の定期更新

**目的**: 専門家としての資格証明、HPのアクティブ感演出

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| 定期更新 | 週1回以上の更新頻度 | 🔴 高 |
| トップページ表示 | 最新記事をHPトップに掲載 | 🔴 高 |
| 専門的知見 | 業界トレンド、ノウハウの発信 | 🔴 高 |
| 最終更新日表示 | 「半年前」ではないことを明示 | 🔴 高 |

**実装ページ**: `/articles` および `/articles/[slug]`

**既存実装**: `content/articles/*.md` で基盤あり

**追加実装**:
- トップページに「最新記事」セクション追加
- 記事一覧ページの充実
- 更新日の目立つ表示

### 7. コラムタイトルを大手メディアに合わせる

**目的**: メディアが求めるテーマとのマッチング

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| タイトル最適化 | Yahoo!ニュース等で読まれている記事タイトルを参考に | 🟡 中 |
| キーワードリサーチ | 業界キーワードでの検索結果分析 | 🟡 中 |
| GEO連携 | GEOシステムで可視性の高いテーマを特定 | 🟡 中 |

**運用プロセス**:
1. 業界キーワードでGoogle/Yahoo!ニュース検索
2. 上位表示されている記事のタイトルパターン分析
3. 同様のテーマで自社視点の記事を作成
4. GEOでvisibility評価し、改善ループ

---

### 8. プレスリリース（ニュース）の集約と強調

**目的**: オフィシャル感と権威性の付与、検索結果での露出

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|
| ニュース一覧 | HP最上部（バナー下）にリスト表示 | 🔴 高 |
| サムネイル | 視覚的に豪華なデザイン | 🟡 中 |
| PRプラットフォーム連携 | PR TIMES等へのリンク | 🟡 中 |
| RSS/自動取得 | プレスリリースの自動表示 | 🟢 低 |

**実装ページ**: `/news` および トップページ

**データ構造案**:
```typescript
interface PressRelease {
  id: string
  title: string
  summary: string
  thumbnail?: string
  externalUrl?: string       // PR TIMES等へのリンク
  publishedAt: string
  category: 'service' | 'partnership' | 'award' | 'event' | 'other'
}
```

**目的**: TVドラマ・映画等での受動的メディア露出機会の創出

| 項目 | 実装内容 | 優先度 |
|------|----------|--------|

| 写真ギャラリー | 撮影に適した空間の写真 | 🟢 低 |
| 問い合わせ導線 | ロケ利用専用の問い合わせ | 🟢 低 |

**実装ページ**: `/press` 内のセクション または `/location`

## III. 避けるべき点チェックリスト

| チェック項目 | 確認方法 | 対策 |
|-------------|----------|------|
| 最終更新が半年以上前 | トップページの日付確認 | 週1回以上の記事/ニュース更新 |
| 経歴が曖昧な表現 | 「様々な...」の有無確認 | 具体的な固有名称に書き換え |
| メディア導線なし | `/press`ページの有無 | メディア専用ページ作成 |
| 実績の客観性不足 | 自己アピールのみになっていないか | 顧客事例・第三者評価の追加 |
