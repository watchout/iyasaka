## 1. プロダクト概要

### 1.1 プロダクト名

IYASAKA Web（コーポレートサイト + リード獲得プラットフォーム）

### 1.2 ミッション

「今ある『不』を、未来の『光』へ。」

中小企業の現場に潜む「不」（痛み・課題）を、IYASAKAの5つの「プラス」製品で「光」（解決・繁栄）へ変換する。
Webサイトはその接点（タッチポイント）として、共感 -> 診断 -> 相談 -> 導入の導線を担う。

### 1.3 プロダクトビジョン

Phase 1: IYASAKAのコーポレートサイトとして営業ツール + リード獲得基盤を構築
Phase 2: マルチテナントHP SaaS化（東海電業社を再現性検証テナントとする）
Phase 3: 業種パック化・AI生成・課金連携で製品化

### 1.4 技術スタック

| 項目 | 選定 |
|------|------|
| フレームワーク | Nuxt 3 (Vue 3.5) |
| スタイリング | Tailwind CSS 3.4 |
| データベース | Supabase (PostgreSQL) |
| ホスティング | VPS (ConoHa Ubuntu 22.04, Nginx) |
| 計測 | Plausible Analytics |
| メール | さくらSMTP (nodemailer) |
| 開発ポート | 4100 |
| 本番ドメイン | iyasaka.co |

## 3. ターゲットユーザー

### 3.1 プライマリペルソナ

**「IT担当者不在の中小企業経営者」**

| 属性 | 内容 |
|------|------|
| 業種 | 電気工事・設備、建設・製造・物流、ホテル・旅館、小売 |
| 規模 | 従業員3-100名 |
| IT環境 | 担当者不在、Excel/ホワイトボード中心 |
| 課題 | 現場の不透明、設備トラブル、人手不足、DXの形骸化 |
| 行動 | Web検索 -> 「自社に合うソリューション」を探す |
| 決裁 | 経営者本人が判断（コスト感: 月額数万-十数万円） |

### 3.2 セカンダリペルソナ

**「兼務マーケ担当者」**

| 属性 | 内容 |
|------|------|
| 規模 | 従業員50-200名 |
| IT環境 | 基本ツールは導入済みだが活用不十分 |
| 課題 | 専門開発リソースがない、ツールが定着しない |

## 4. ユーザーストーリー

### 4.1 MUST（P0）

| ID | ユーザー | やりたいこと | 理由 |
|----|---------|------------|------|
| US-01 | 経営者 | トップページでIYASAKAの価値を理解したい | 信頼できるパートナーか判断するため |
| US-02 | 経営者 | 自社の課題に合う製品を見つけたい | どの「プラス」が最適か知りたいため |
| US-03 | 経営者 | AI活用診断で自社の状況を客観視したい | 何から始めればよいか分からないため |
| US-04 | 経営者 | 診断結果から無料相談に申し込みたい | 具体的な提案を受けたいため |
| US-05 | 経営者 | 問い合わせフォームから相談したい | 電話より気軽に始めたいため |
| US-06 | 経営者 | スマホでもストレスなく閲覧したい | 移動中や現場で見るため |

### 4.2 SHOULD（P1）

| ID | ユーザー | やりたいこと | 理由 |
|----|---------|------------|------|
| US-07 | 経営者 | 導入事例を見て安心したい | 同業他社の成功を確認したいため |
| US-08 | 経営者 | 記事コンテンツで業界知識を得たい | 情報収集しながら信頼を蓄積するため |

## 7. 制約・前提条件

### 7.1 技術的制約

| 制約 | 詳細 |
|------|------|
| SSR必須 | AIEO対応のため、サーバーサイドレンダリングが必要 |
| VPSホスティング | ConoHa VPS（Vercelではない）、PM2で稼働 |
| `window` API禁止（SSR） | ブラウザAPI使用時は `import.meta.client` ガード必須 |
| treeshake annotations無効 | Vue 3.5 + Nuxt 3.10 のSSR対策、削除禁止 |

### 7.2 ビジネス的制約

| 制約 | 詳細 |
|------|------|
| 農業は現行サービスに含めない | 「未来の展望」としてVisionセクションでのみ言及 |
| ブランド一貫性 | 全ページでナラティブOS（不 -> 光）を貫通 |
| CTA統一 | `https://iyasaka.co/contact?p_id=[プロダクトID]` 形式 |

### 7.3 運用的前提

| 前提 | 詳細 |
|------|------|
| Phase 1ではDB保存なし | リードはメール通知のみで運用（Phase 2でSupabase連携） |
| デプロイ | `scripts/vps-deploy.sh` による手動デプロイ |
| SSL | Let's Encrypt (certbot) 自動更新 |
