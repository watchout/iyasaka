# 事業計画（3〜6ヶ月）

## 0. 事業の定義（1文）

現場の「不」を、診断→設計→導入→運用で「光」に変える、再現性のある伴走型DX支援。

## 0.5 事業内容（現行）

経営上の判断やKPI設計で迷ったら、まずここに戻る（増やす/減らす/統合するの判断軸）。

### 事業ライン（6つ）

- **弱電事業（現行の基盤）**  
  - 提供: 弱電工事・電気工事（必要な改修/増設/是正を含む）  
  - 価値: 「止まる前に手を打つ」ための現場の足元を整える  
  - 収益: 工事（スポット） + 保守（継続化できる場合はストック化）

- **ホテルAI（客室→PMS→CRM）**  
  - 提供: 客室AIコンシェルジュ / AIホテルPMS / AIホテルCRM  
  - 価値: 人手不足下でも接客品質と運用効率を両立し、再来までつなぐ  
  - 収益: 初期導入 + 月額（運用・改善）

- **配信プラス（JV/外注/スクール）**  
  - 提供: 動画配信のJV事業 / 外注 / スクール（内製化）  
  - 価値: 投資リスクを抑えながら、配信を収益源に変える  
  - 収益: 売上シェア（JV）/ 受託（スポット・月額）/ スクール

- **ミエルプラス（ボード/ストック/ドライブ）**  
  - 提供: 現場をDXとAIで見える化（進捗/在庫/資料）  
  - 価値: 探す・確認する時間を減らし、現場判断を速くする  
  - 収益: 月額（運用定着） + 必要に応じた初期導入

- **AIプラス（カスタムAI構築・保守）**  
  - 提供: カスタム型AIシステムの構築と保守  
  - 価値: 既製品では解けない固有課題を、止まらない形で実装する  
  - 収益: 受託（個別見積） + 保守（月額）

- **dev-OS（SSOTベースAI開発SaaS）**  
  - 提供: SSOTを軸にしたAI開発の監査・品質・運用を支えるSaaS  
  - 価値: 仕様の揺れ/手戻り/事故を減らし、開発の再現性を上げる  
  - 収益: 月額（将来は外販も可）

## 1. 対象（誰のどんな不を解くか）

### 優先ターゲット（3ヶ月）

- 中小企業（従業員3〜100名規模）
- 現場責任者/経営者（IT担当がいない、または兼務）

### 痛み（不）の入口（共通）

- 見えない/把握できない（現場の状況）
- 止まったら終わり（インフラ）
- 人手不足で品質が落ちる（接客/対応）
- 収益の多角化ができない（投資リスクが怖い）
- DXツールが定着しない（形骸化）

## 2. 提供価値（何を約束するか）

3ヶ月は「各プロダクトを売る」より、**次の合意（無料先行予約）**を取りにいく。

### プレローンチの提供物（最小）

詳細は `PRELAUNCH_PLAYBOOK.md` に集約。

- 現状ヒアリングの要点（A4 1枚）
- 90日間の打ち手（優先順位付き）
- 先行導入（または先行設計）枠の仮押さえ（無料先行予約）

## 3. 収益モデル（6ヶ月で固める）

※3ヶ月は検証優先。6ヶ月で「勝ち筋の収益モデル」を固定する。

- サブスク（保守/運用）: 弱電プラス等
- 月額（運用伴走/可視化）: ミエルプラス等
- 収益シェア（JV）: 配信プラス等
- 個別見積（開発）: AIプラス等

## 4. 集客・販売（Go-to-Market）

### 優先順位（3ヶ月）

1. 紹介（既存ネットワーク・協力会社）
2. 提携（JV/施工/運用パートナー）
3. アウトバウンド（業種×痛みで絞る）
4. サイト/コンテンツ（信頼形成 + 刈り取り）

### 多角化に関する方針（プレローンチ期）

3ヶ月は「新商品×新市場（多角化）」に走るほど失敗確率が上がるため、原則避ける。  
既存の強みを活かし「既存商品×新市場」「新商品×既存市場」を優先する。

### オファーの作り方（根拠フレーム）

- 神田昌典（PASONA）: 問題→共感→解決→提案→絞り込み→行動  
  参考: `https://www.satorifactory.jp/%E3%80%90%E3%83%9E%E3%83%BC%E3%82%B1%E3%82%BF%E3%83%BC%E5%BF%85%E8%A6%8B%E3%80%91%E7%A5%9E%E7%94%B0%E6%98%8C%E5%85%B8%E3%80%8E%E7%A8%BC%E3%81%90%E8%A8%80%E8%91%89%E3%81%AE%E6%B3%95%E5%89%87%E3%80%8F/`
- ジェイ・エイブラハム（売上を増やす3つ）: 顧客数/単価/頻度  
  参考: `https://blog.myasp.jp/increase-sales-per-customer/`
