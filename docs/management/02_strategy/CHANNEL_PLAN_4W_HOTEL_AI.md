# 4週間 チャネル計画（ホテルAI：紹介＋広告）

このドキュメントは、12週間の主戦場（ホテルAI）を崩さずに「紹介」と「広告テスト」を回すための運用計画（SSOT）です。

## 0) 固定（この4週間は変えない）

- 主戦場: ホテルAI（入口: 客室AIコンシェルジュ）
- ターゲット: 小規模ホテル（〜50室）
- エリア: 関東中心
- 北極星（経営判断）: FreeReservation（次回合意）
- 予算上限（広告）: 5万円 / 4週間

## 1) 紹介（最優先チャネル）

### 週次の最低ライン（目安）

- 紹介依頼: 10件/週
- フォロー: 10件/週

### 紹介依頼メッセージ（雛形）

- 目的: 「小規模ホテル（〜50室）で、深夜・定型問い合わせ対応に困っている方」を1名だけ紹介してほしい
- オファー: 7日トライアル（FAQ + 受付 + 多言語）で「スタッフ対応時間の削減（分/日）」を検証

## 2) 広告（最小テスト）

### 運用方針（重要）

- 広告プラットフォームの最適化は、中間CV（ContactSubmit/FirstCallScheduled等）も併用する
- 経営判断は FreeReservation で行う（数が少ないうちは特に）

### テスト設計（4週間）

- Week 1: キーワード/広告文の初期セット → 反応の良い軸を見つける
- Week 2: 除外KW + LP/広告文を1回だけ改善
- Week 3: “刺さった軸” に予算集中（分散しない）
- Week 4: 予約に近い導線へ寄せる（初回商談の質を上げる）

## 3) 計測（最低限）

- ContactSubmit（フォーム送信）
- FirstCallScheduled（初回商談予定）
- FirstCallHeld（初回商談実施）
- FreeReservation（次回合意）

※入力は `WEEKLY_SCORECARD.md` と `DAILY_CHECKIN_TEMPLATE.md` に統一する
