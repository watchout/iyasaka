# ホテルAI オファー仕様（12週間の正本）

このドキュメントは、ホテルAI（客室AIコンシェルジュ）を「売れる形」に固定するための仕様書です。  
12週間は、この仕様を基本として運用し、週次で更新します。

## 1) 主戦場（固定）

- 事業: **ホテルAI**
- 入口（楔）: **客室AIコンシェルジュ**
- ターゲット: **小規模ホテル（〜50室目安）**
- エリア: **関東中心**

## 2) 入口オファー（7日トライアル）

### 目的

- 深夜・定型問い合わせを減らし、**スタッフ対応時間を削減**する
- 「導入して終わり」ではなく、運用で回る形（フォールバック含む）を作る

### 提供範囲（確定）

- **FAQ**: 館内/周辺/手続き（チェックイン/アウト、設備、アクセス等）
- **受付**: 要望/備品/簡易オーダー等を受け付けて現場へ回す
- **多言語**: 英語・必要に応じて中/韓など（比率に合わせて調整）

### フォールバック（止めない）

- 受付は「通知/起票/手動連携」でも開始し、運用の詰まりを先に潰す
- PMS連携は任意（未連携でも開始できる）

### 成功KPI（最重要）

- **スタッフ対応時間の削減（分/日）**

## 3) 先行予約（FreeReservation）で約束するもの

`PRELAUNCH_PLAYBOOK.md` に準拠し、次の3点を固定する。

- 現状ヒアリングの要点（A4 1枚）
- 90日間の打ち手（優先順位付き）
- 先行導入（または先行設計）枠の仮押さえ

## 4) 有料プラン（形だけ先に固定）

数値（料金）は **先行告知済みのため、この仕様を正本として固定**する。  
誤りがあった箇所は、ここで修正して統一する（口頭で揺らさない）。
