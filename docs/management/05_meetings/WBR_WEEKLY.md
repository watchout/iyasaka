# WBR（週次経営レビュー）

## 目的

- 数字と行動を分けて見て、改善速度を最大化する
- 今週やること/やらないことを固定し、ブレを減らす

## 役割（最小）

- 進行（Facilitator）: 会議の時間管理、結論の確定
- 記録（Scribe）: スコアカード/実行ログ/決定事項の更新
- オーナー（Owner）: 今週の重点の最終責任（やらないことの確定を含む）

## 事前準備（必須）

- `04_scorecards/WEEKLY_SCORECARD.md` を更新してから実施する
- `07_execution/EXECUTION_LOG.md` に今週の実行と結果を記録する

## タイムボックス（推奨 45分）

1. 先週の結果（10分）: スコアカード確認（事実のみ）
2. ギャップ（10分）: 目標との差分が大きい箇所を特定
3. 原因仮説（10分）: 何が原因か（仮説でよい）
4. 今週の打ち手（10分）: 最大3つに絞る（やらないことも決める）
5. 担当/期限（5分）: 担当者・期限・成否判定（KPI）を確定

## 意思決定ルール

- 施策は「最大3つ」まで。増やすほど実行が薄まる
- 迷ったら `01_vision/VISION.md` の判断フィルターに戻る
- 定義が揉めたら `03_kpi/KPI_DICTIONARY.md` を更新して揃える
