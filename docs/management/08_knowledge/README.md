# 知識データベース（経営・仕組み化）

## 目的

成功者のノウハウを「知識のまま」で終わらせず、IYASAKAの運用に **取り入れて回せる形（SOP/チェックリスト/KPI）** に変換する。

## 基本方針（ノウハウ → 仕組み）

1. **リサーチする（Input）**
2. **要約する（Distill）**: 5原則 + 3注意 + 1テンプレ
3. **仕組み化する（Systemize）**: SOP化、チェック項目化、会議体に組み込む
4. **計測する（Measure）**: KPI辞書に定義、スコアカードで追う
5. **更新/破棄する（Update/Drop）**: 効かないものは捨てる

## ディレクトリ構成

- `00_inbox/`: 収集したリンク、メモ、未整理のノート（まず溜める）
- `01_cards/`: 知識カード（要約済み、再利用可能）
- `02_sops/`: SOP（手順書、運用の型）

## 1本の動画/記事から作る成果物（標準）

最低限、次のどれか1つに落とす（全部やろうとしない）。

- 知識カード（`01_cards/*`）
- SOP（`02_sops/*`）
- KPI辞書の更新（`../03_kpi/KPI_DICTIONARY.md`）

## “闇雲”を防ぐルール

- **毎週の研究テーマは最大1つ**（並行しすぎない）
- **今週の運用に直結しない研究は後回し**（3ヶ月は特に）
- **結論は必ず「何をいつ誰がやるか」に変換**（SOP化できないなら保留）

## 使い方（毎日）

1. その日に見た動画/記事を `00_inbox/` に追加（リンクとメモでOK）
2. 重要だと思ったものを「知識カード」にする（テンプレは `KNOWLEDGE_CARD_TEMPLATE.md`）
3. カードが運用に効くならSOP化して `02_sops/` へ
4. 週次WBRで「採用/保留/破棄」を決める
