# パイプライン定義（Lead → 無料先行予約 → 契約）

## 目的

- 「どこまで進んだら予約なのか」を固定し、数がブレないようにする
- 週次WBRでボトルネックを特定できるようにする
- 対象: **ホテルAI（入口: 客室AIコンシェルジュ / ターゲット: 小規模ホテル）**

## ステージ（推奨）

| ステージ | キー（例） | 定義 | 次のアクション |
|---|---|---|---|
| Lead | lead | 連絡先が取れた見込み顧客（フォーム/紹介/アウトバウンド） | 24h以内に一次返信/初回打診 |
| Contacted | contacted | 接触が完了（返信/通話/DM）し、課題の一次情報が取れた | 適合判定（Qualifiedへ） |
| QualifiedLead | qualified | 痛みが明確で、提案価値があり、次回商談に進める状態 | 初回商談を設定 |
| FirstCallScheduled | first_call_scheduled | 初回商談の日時・参加者が確定 | 実施（Heldへ） |
| FirstCallHeld | first_call_held | 初回商談を実施し、次の合意に進める状態 | 予約打診（Reservationへ） |
| FreeReservation | free_reservation | 無料先行予約（仮押さえ）が成立 | 次回（設計/現地/資料）を確定 |
| Converted | converted | 有料契約に移行（または契約同等の発注確約） | オンボード/実装 |
| Lost | lost | 当面見送り/失注 | 理由を記録し、再接触条件を設定 |

## FreeReservation（無料先行予約）の判定基準

次の3点が揃ったら FreeReservation とする。

1. 次回アクションが確定（日時/参加者/目的）
2. 現状情報の提出に合意（例: 現状運用、設備、体制、既存ツール）
3. 優先度が明確（期限/緊急度/背景）

補足（ホテルAI・客室AIコンシェルジュ）:

- 次回アクション例: 「7日トライアル開始のキックオフ」「館内/周辺QA素材の受領」「運用フロー確認」

「温度感はあるが未確定」は QualifiedLead に留める。

## 失注/保留理由（カテゴリ）

運用で増えるほど改善が速くなるため、分類を固定する。

- タイミング不一致（今は動けない）
- 予算が合わない/不明
- 意思決定者不在
- 競合に決定
- 痛みが弱い/優先度低
- 要件が未整理（設計が先）
- 信頼不足（実績/不安）

## 運用ルール（SLA）

- Lead受領 → **24時間以内** に一次返信（遅れるほど失注率が上がる）
- FirstCallScheduled → 実施前に「目的/宿題/アジェンダ」を送付
- Lost → 再接触条件（いつ/何が揃えば）を必ず残す

## 記録方法（どこに残すか）

まずは「回す」ことを優先し、次のどちらかに統一する。

1. Supabase `leads.status` を更新（ダッシュボード運用）
2. `04_scorecards/WEEKLY_SCORECARD.md` のパイプライン欄で手入力（数だけでも可）
