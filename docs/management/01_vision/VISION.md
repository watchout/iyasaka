# ビジョン（経営版SSOT）

このドキュメントは、経営判断に必要な「言葉の固定」を行います。  
ブランドの正本は `docs/ssot/BRAND_MASTER_PROTOCOL.md` と `docs/ssot/BRAND_NARRATIVE.md` にあります。本書は **経営運用で使える形** に要点を抜き出します。

## ミッション（Mission）

今ある「不」を、未来の「光」へ。

## ビジョン（Vision）

**世の中のあらゆる「不」を、技術で解消する。**

### ビジョンの定義（言葉の固定）

- **「不」**: 止まる / 見えない / 属人化 / 続かない / 収益化できない（投資リスクが怖い）
- **「技術」**: 弱電・電気工事（現場の足元） + ソフト/AI + 運用設計（定着） + SSOT（仕様の正本）
- **「解消」**: 現場の時間・不安・ミスが減り、判断が速くなり、挑戦できる状態が増える

### 弥栄（いやさか）としての到達状態

- 顧客・利用者・地域が等しく潤う「弥栄の循環」を実装する

## 事業内容（現行ポートフォリオ）

- **弱電事業**: 弱電工事・電気工事（現行事業）
- **ホテルAI**: 客室AIコンシェルジュ / AIホテルPMS / AIホテルCRM
- **配信プラス**: 動画配信のJV事業 / 動画配信の外注事業 / 配信スクール事業
- **ミエルプラス**: 現場をDXとAIで見える化（ミエルボード / ミエルストック / ミエルドライブ）
- **AIプラス**: カスタム型AIシステム構築・保守
- **dev-OS**: SSOTベースAI開発SaaS

## バリュー（Values）

- 現場第一主義: 綺麗な理屈より、現場の痛みを先に消す
- プラスの精神: 既存の努力を壊さず、そこに技術を足す
- 弥栄の循環: 一社勝ちではなく、関わる全員が潤う設計を優先する

## 3つの変換動詞（事業の分類）

- 整える（Foundation）: 足元を整え、確信をつくる（弱電プラス、ミエルプラス、dev-OS）
- つなぐ（Expansion）: 分断をつなぎ、可能性を拡張する（配信プラス、多言語プラス）
- 育てる（Evolution）: 導入で終わらず、運用で文化を育てる（OmotenasuAI、AIプラス）

## 経営判断のフィルター（迷ったらここに戻る）

次の問いに、Yesが多いほど優先度は高い。

1. 顧客の「不」を具体的に減らすか（現場の時間・不安・ミス）
2. 期待する「光」が測れるか（数値・行動の変化）
3. 「プラスのUI/運用」になっているか（既存ワークフローを壊さない）
4. 継続できるか（再現性・運用負荷・サポート体制）
5. 弥栄の循環に寄与するか（顧客/利用者/地域に波及するか）

## 制約（やらないこと）

- 農業は **現在のサービスラインナップに含めない**（未来の展望として静かに言及するに留める）
- リード獲得は **公式サイトに集約**（LP側で独自フォームを作らない）
